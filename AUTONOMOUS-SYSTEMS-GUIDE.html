Core Philosophy

▼
The Fundamental Rule
Engines for math, Claude for prose. Deterministic Node.js engines handle data processing (API calls, scoring, diffing) at zero token cost. Claude agents read engine output and handle judgment calls (drafting, synthesis, strategy).

AI Employee Team

Design Principles

Autonomous discovery, human-approved actionSystem finds opportunities and proposes improvements. David approves or rejects.
Evidence-based, not vibes-basedEvery opportunity traces to a data signal. Every proposal cites feedback data.
Reversible by defaultProposals go to staging directories. Prompt changes are versioned. Nothing permanent without approval.
Tier-based autonomyTier 0: observe. Tier 1: draft. Tier 2: needs approval. Tier 3: always blocked. System earns trust through demonstrated competence.
Self-improvement through ReflexionEvery agent self-reflects in handoffs. OPTIMIZER tracks quality trends. The system gets smarter over time.

The Handoff Protocol

Every agent session produces a structured handoff file at assistant/state/handoffs/{date}-{agent}.md containing:

1. WORK COMPLETEDWhat was done this session
2. TASKS FLIPPEDWhich goals were advanced
3. BLOCKERSWhat prevented more progress
4. NEXT SESSIONRecommended work for next heartbeat
5. FOR DAVIDItems requiring human action (max 3)
6. SELF-REFLECTIONQuality score 1-5 with evidence (Reflexion pattern)

Self-Improvement System

AUTONOMOUS LEARNING ▼

Three Feedback Loops

LoopFrequencyHow It Works
Reflexion (agent self-review)Every sessionEach agent rates its own output 1-5 with evidence. OPTIMIZER tracks trends. Declining quality triggers prompt review.
Task Feedback (David's ratings)Via /standup, /rate-overnightDavid marks tasks as used/skipped/reworked. Utility rate tracked per engine, agent, and signal type. Target: >60%.
Monthly CalibrationVia /calibrateOPTIMIZER proposes threshold changes based on 30-day feedback data. David approves/rejects each change.

Tier System & Safety

▼
TierAutonomy LevelExamples
Tier 0Observe/report only - writes to assistant/state/ onlyHealth alerts, staleness reports, metric anomaly analysis
Tier 1Draft to file - creates content but nothing externalEmail drafts, outreach copy, blog articles, analysis reports
Tier 2Requires explicit approval - David must approveEmail sends, campaign changes, Shopify updates, financial writes
Tier 3Always blocked - never autonomousTheme push to live, customer-facing changes, spending money
Trust Building
The system earns autonomy through demonstrated competence. When task utility rate consistently exceeds 75% for a signal type, OPTIMIZER proposes promoting related tasks from Tier 1 to Tier 2 (with David's approval). Current utility rate: 63% - not yet at the promotion threshold.

Safety Mechanisms

Single-process lock~/.config/aware-hair/locks/claude.lock prevents concurrent Claude sessions
Per-agent locksEach agent has its own lock file, stale after 900 seconds
Hard deadlineOvernight dispatch stops at 04:30 AM to free lock for morning tasks
Auth pre-check5-second token validation before any Claude call (fast-fail)
Pause mechanism/pause-automation creates a flag file that all cron tasks check
Max budgetPipe-mode sessions use --max-budget-usd to cap costs
Max turnsAgents limited to 25 turns, overnight tasks to 50 max
Tool restrictionsAgents only get Read,Write,Edit,Grep,Glob,WebSearch,WebFetch
