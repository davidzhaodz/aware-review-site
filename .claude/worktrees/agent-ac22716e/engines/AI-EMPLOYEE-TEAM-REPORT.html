The Architecture

The system splits work between deterministic Node.js engines (math, API calls, scoring) and Claude Code sessions (judgment, drafting, synthesis). Engines run in seconds for $0. Claude is reserved for what it's good at: prose and decisions.

Engines for math. Claude for prose.

The reorder-prophet script previously used Claude to run curl commands and calculate division - costing ~$0.15 per run for deterministic arithmetic. SENTINEL now does the same work in 3 seconds for $0. Across all three engines, we eliminated 4 Claude invocations per day, saving ~$0.50/day and 20+ minutes of execution time. Claude is now only invoked for judgment calls: drafting reactivation emails, synthesizing competitor intelligence, and generating content briefs.

The Replicable Engine Pattern

Every engine follows a 7-step contract. To add a new business function (email health, finance, etc.), implement these 7 steps with domain-specific data sources and scoring formulas.
1

Data Pull

API calls to source systems using fetchJSON() or fetchAllPages() from engine-base.mjs. Handles pagination, rate limiting, and retries automatically.

2

Filter

Domain-specific criteria. SENTINEL: B2B customer tags. PUBLISHER: 7+ word queries. SCOUT: page-level content extraction. Reduces raw data to actionable records.

3

Cluster

Group by shared attributes. SENTINEL: orders by salon. PUBLISHER: queries by shared keywords. SCOUT: changes by competitor. Transforms individual data points into business entities.

4

Score

Quantify opportunity or risk on a single axis. Formula pattern: volume x performance x room-for-improvement. SENTINEL uses days_since / avg_interval. PUBLISHER uses impressions / position.

5

Context Check

Compare against previous run via loadPreviousRun(). Detects what's new (new RED salon), what resolved (salon ordered), and what worsened. Prevents alert fatigue by highlighting changes.

6

Brief Generation

Standardized markdown via writeBrief(). Three sections: ALERT (immediate), WATCH (trending), HEALTHY (on track). Written to ~/.config/aware-hair/briefs/YYYY-MM-DD/.

7

Signal Emission

Write to shared signal-results.json via emitSignal(). Each signal has: engine, name, severity, human-readable value, and machine-readable details object. Task generator reads this at 20:00.
