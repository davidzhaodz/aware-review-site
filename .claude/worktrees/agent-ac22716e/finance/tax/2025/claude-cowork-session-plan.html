Decision Framework: What Action to Take

For every bank feed item, there are exactly 4 possible actions. The correct action depends on whether the transaction already exists in QBO.

For each pending bank feed item, follow this tree: 1. Does QBO show a green "MATCH" indicator? YES -> Verify the matched transaction (date within 2 days, amount within $0.01) -> If match looks correct: click MATCH -> If match looks wrong: SKIP - flag for investigation NO -> Continue to step 2 2. Is this a transfer between two AWARE accounts? (CC payment, Shopify->Checking, internal transfer) YES -> Click TRANSFER, select the correct destination/source account -> IMPORTANT: Only process the SENDING side. The receiving side should auto-match. NO -> Continue to step 3 3. Is the vendor in the rules table with an explicit category? YES -> Does QBO's suggestion match the rule? -> YES: Click ADD (accept QBO suggestion) -> NO: Override to the correct category, then click ADD NO -> SKIP - do not guess, do not accept QBO's suggestion
CRITICAL ORDER: Always check for Matches FIRST. If you click "Add" on an item that should have been "Match," you create a duplicate transaction that double-counts the expense or income. This is the #1 most damaging error in bank feed processing.

Understanding Add vs Match vs Transfer

ActionWhen to UseWhat QBO DoesIf Misused
MATCH Transaction already exists in QBO (invoice payment, A2X journal entry, manually entered expense) Links the bank item to the existing record. Marks it "cleared." If wrong match: ledger entry points to wrong transaction
ADD Transaction does NOT exist in QBO yet. This is a brand new expense or deposit. Creates a new transaction in the ledger from the bank data. DUPLICATE RISK If transaction already exists, creates double entry
TRANSFER Money moving between your own accounts (CC payment, Shopify payout to checking, internal transfer) Records a debit in one account and credit in the other. If "Add" used instead: creates phantom income or expense
SKIP Unknown vendor, unusual amount, or not in rules table Nothing. Item stays in "For Review." No risk. Can always come back to it.

What Claude Should Never Do

  • Never invent account names. Only use account names confirmed to exist in QBO. If unsure, say "verify this account exists in the QBO dropdown before selecting."
  • Never state a debit/credit direction without explanation. Always clarify: "This is an expense (debit to Rent, credit to Checking)" or "This is a liability (credit to Loan Payable)."
  • Never recommend "Add" without first checking if a Match might exist. Always say "Check for a Match indicator first."
  • Never provide tax advice. Category recommendations are for bookkeeping purposes. Tax treatment decisions are for the CPA.
  • Never present estimates as facts. If pulling QBO data, label it [LIVE: QBO date]. If from memory, label it [APPROXIMATE].
