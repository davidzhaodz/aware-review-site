Cost model: All agents run on Claude Max subscription via claude -p. Zero API cost. The engines are Node.js (zero tokens). The entire system runs at the cost of the Max subscription - no per-token billing.

1. Design Principles

"Engines for math, Claude for prose"

Deterministic engines (Node.js, zero tokens) handle data pull, scoring, diffing, and alerting. Claude agents handle judgment, writing, and synthesis. The supervisor daemon handles scheduling and process management. Each layer is more reliable than the one above it.

"The system proposes, David disposes"

Agents draft, recommend, and analyze. They never send emails, publish content, make purchases, or commit to external parties. All customer-facing output goes through David's review. Trust zones define what can auto-execute vs. what needs approval.

"Idle capacity should always be working"

When no scheduled agent is running, the daemon dispatches proactive work from the idle queue: prospecting, content drafting, competitive research, and infrastructure maintenance. Tiered by business value.

"Institutional knowledge is the deepest moat"

Every session generates structured reflections. Lessons promote from candidate to proven over time. The system compounds knowledge about salons, competitors, and what works - data no competitor can replicate.

Weekly MAR Review (Saturday, CHIEF-led)

Multi-Agent Reflexion debate - structured cross-agent knowledge review:

  1. Read BEACON digest (includes 5-metric behavioral health check)
  2. Collect all agent reflections from assistant/state/reflections/
  3. Cross-examine: contradictions? Cross-domain insights? Promotion-ready?
  4. Synthesize: promote up to 5 lessons, flag contradictions for David
  5. Archive processed reflections
